Real-Time Anomaly Detection Pipeline

6 microservices · Kafka · Redis · TimescaleDB · Prometheus · Grafana

PRODUCER
Generates a fake financial transaction every 10ms Phase 1 (Normal) → Phase 2 (Drift) → Phase 3 (Attack) → repeat
↓
Kafka: raw-transactions
↓
FEATURE ENGINEER
Looks up sliding-window stats from Redis (averages, velocity, risk flags) Attaches computed features to the event
↓
Kafka: enriched-features
↓
ML INFERENCE
Scores each transaction from 0.0 (normal) to 1.0 (anomalous) Updates the model instantly after scoring (online learning) Flags event as anomaly if score > dynamic threshold Auto-adjusts threshold to keep anomaly rate between 0.1% – 5%
↓
Kafka: anomaly-scores
↙
↘
DRIFT DETECTOR
Watches for shifts in feature distributions (ADWIN) Watches for concept drift in error rate (DDM) 300s cooldown between triggers
ALERT ROUTER
Filters events where is_anomaly = true Deduplicates via Redis TTL Buckets into LOW / MEDIUM / HIGH / CRITICAL Sends Slack webhook for HIGH+
↓
↓
Kafka: retraining-triggers
TimescaleDB: alerts
↓
RETRAINING WORKER
Trains a fresh model on recent data from TimescaleDB Shadow-tests it: runs old and new model in parallel Promotes new model to Redis only if accuracy improves
↓
Redis — new model written to key active_model
↓
ML INFERENCE
Polls Redis every 1000 events for a new model version Hot-swaps the model in-place — zero downtime
All services feed into ↓
TIMESCALEDB
Permanent storage
scored_events, alerts,
drift_events, model_versions
